## ❌ 剩余失败 (3个测试)

### 1. cond_impl (条件实现语法)
**问题：** `with Show<T>` 条件trait约束未实现  
**示例：**
```glue
type Box<T>: Show = Box(value: T) with Show<T> {
    fun show(self): str { "Box(" + self.value.show() + ")" }
}
```
**性质：** 待实现特性  
**工作量：** 2-3周（解析器 + 类型检查器）

### 2. edge_records_traits (ADT 多trait)
**问题：** ADT 不能实现多个 trait  
**示例：**
```glue
type X: Trait1, Trait2 = | A | B {
    // ... 实现方法
}
```
**根因：** `checkNoConflictingImpls` 返回 Unsupported  
**验证：** 单个 trait 可工作，多个 trait 失败  
**性质：** 待实现特性  
**工作量：** 1-2周（实现trait组合分派）

### 3. test_module_trait (文件系统模块)
**问题：** 模块加载器只支持 stdlib，不支持文件系统  
**性质：** 待实现特性  
**工作量：** 2-3天

## 🎯 关键发现和洞察

### 1. 运算符实现的系统性问题
发现了两个严重的运算符bug：
- 所有整数比较运算符执行相同的 `==` 检查
- `!=` 运算符未对结果取反

**教训：** 运算符实现缺乏系统性测试和代码审查

### 2. Nullable 类型细化已正确实现
类型检查器的 `applyNarrowing` 设计良好，只是被 `!=` bug 阻塞。修复 `!=` 后立即工作。

**教训：** 核心类型系统是健壮的

### 3. 语法糖 vs 完整语法
- `Error("msg")` 简写语法未实现
- `with Show<T>` 条件实现未实现
- 但完整语法都正常工作

**教训：** 许多"失败"是语法糖问题，不是核心功能bug

### 4. ADT 多trait限制 ⭐
通过系统性诊断发现：
- 单个 trait: `type X: Trait = ...` ✓
- 多个 trait: `type X: T1, T2 = ...` ✗

**教训：** 这是编译器的已知限制

### 5. 连锁反应效应
修复 `!=` 和 `op_test_lit` 两个bug后，4个测试立即通过。

**教训：** 核心bug的影响范围广
