## 失败测试详细分析

### 1. nullable-some / nullable-none ✓ 已修复

**问题**: 测试使用了错误的Nullable模式匹配语法

**错误代码**:
```glue
val x: i64? = 42i64
match x {
    Some(v) => println(v),  // ✗ 错误：Nullable不使用Some
    None => println("none")
}
```

**正确语法** (language-design.md §2.3.10):
```glue
val x: i64? = 42i64
match x {
    null => println("none"),  // ✓ 直接匹配null
    v => println(v)            // ✓ 直接绑定值
}
```

**文档依据**: §2.3.10明确说明"null作为模式直接匹配"，不使用Option<T>的Some/None包装。

**状态**: 测试用例错误，需要修正

---

### 2. pattern-tuple-destruct ✓ 已确认

**问题**: Glue不支持匿名元组

**错误代码**:
```glue
val pair = (1i64, 2i64)    // ✗ 错误：匿名元组不存在
val (a, b) = pair
```

**文档依据** (language-design.md §2.5.2):
> **无匿名元组**：记录必须有命名字段，`(str, i32)` 不存在。如果需要通用配对类型，使用：
> ```glue
> type Pair<A, B> = Pair(first: A, second: B)
> ```

**正确方式**:
```glue
type Pair<A, B> = Pair(first: A, second: B)
val pair = Pair(1i64, 2i64)
match pair {
    Pair(a, b) => {
        println(a)
        println(b)
    }
}
```

**状态**: 语言设计决策，匿名元组不支持

---

### 3. function-expression-body ⚠️ 语言特性缺失

**问题**: 文档定义了函数表达式体语法，但parser未实现

**文档中的语法** (language-design.md §2.16):
```glue
fun f() = 1
fun f() = 2     // ✗ duplicate definition
```

**当前状态**:
```glue
fun add(a: i64, b: i64): i64 = a + b  // ✗ parse error
```

**workaround**:
```glue
fun add(a: i64, b: i64): i64 { a + b }  // ✓
```

**状态**: 需要在parser中实现表达式体函数支持

**影响**: 低 - 有简单的替代方案（使用块体）

---

### 4. closure-parameter ✓ 已修复

**问题**: 测试使用了错误的函数类型语法

**错误代码**:
```glue
fun apply(f: fun(i64): i64, x: i64): i64  // ✗ 错误语法
```

**正确语法** (language-design.md §2.8):
```glue
fun apply(f: i64 -> i64, x: i64): i64     // ✓ 箭头语法
```

**文档依据**: §2.8明确说明函数类型使用箭头语法：
> 多参数函数类型 `(A, B) -> C` 在类型位置由 `(` ... `)` 后紧跟 `->` ...

**状态**: 测试用例错误，需要修正

---

### 5. map-operations ✓ 已确认

**问题**: Map字面量语法不存在于语言设计中

**错误代码**:
```glue
val m = {"a": 1i64, "b": 2i64}  // ✗ 语法不存在
```

**文档支持的字面量** (language-design.md §2.1.1):
- 整数: `42i64`
- 浮点: `3.14f64`
- 字符: `'a'`
- 字符串: `"hello"`
- 布尔: `true`, `false`
- 数组: `[1, 2, 3]`
- 记录: `(name: "Alice", age: 30)`
- **无Map字面量**

**Map作为标准库模块**:
文档中提到`Collections.Map`作为模块导入，但没有定义字面量语法。

**状态**: 语言设计中未包含Map字面量，测试用例不适用
